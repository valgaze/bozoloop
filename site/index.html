How it works

A goal, an engine, and evaluators.

You bring something that proposes changes and something that judges them. BozoLoop runs the loop, writes every attempt to a ledger, and stops when things pass or you run out of tries.

example.ts 
import { createLoop, commandEvaluator } from "bozoloop";

const loop = createLoop({
  goal: "Make all tests pass",
  workspace: ".",
  maxAttempts: 5,
  engine: {
    async suggest(ctx) {
      // call your LLM, run a codemod, whatever
      return { summary: `fix #${ctx.attempt}`, patch: { /* ... */ } };
    },
  },
  applier: {
    async apply(suggestion) {
      // write files, apply diffs, etc.
      return { success: true };
    },
  },
  evaluators: [
    commandEvaluator("tests", "npm test"),
  ],
});

const result = await loop.run();
// result.status → "completed" | "failed"
Features

Small surface. Real capabilities.

Everything you need for iterative code loops. Nothing you don't.

📒

Full ledger

Every attempt is recorded — patch summary, eval results, pass/fail, timing. Inspect with bozoloop inspect or read .bozoloop/ledger.json directly.

⏸️

Pause & resume

Stop mid-run, come back later. State persists to disk. The ledger continues seamlessly across sessions.

⏪

Checkpoint & rollback

Snapshot your workspace before each attempt. Roll back to the last known good state with one command.

🪝

Lifecycle hooks

Fire callbacks on run start, attempt start/end, suggestion, eval complete, pause, resume, abort, error. Wire up Slack, webhooks, deploy previews.

📦

Zero dependencies

Pure TypeScript compiled to ESM. No runtime dependencies. The whole package is under 15 kB gzipped.

🔧

Typed config

defineConfig() gives you full IntelliSense. bozoloop.config.ts keeps your loop definition clean and version-controlled.

CLI

Run. Resume. Inspect. Rollback.

Four commands. No flags you need to memorize.

$ bozoloop run
Start a new loop from your config file
$ bozoloop resume
Pick up a paused or interrupted loop
$ bozoloop inspect
View status, history, and results
$ bozoloop rollback
Restore the most recent checkpoint
