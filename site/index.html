How it works

Five steps. One loop.

BozoLoop runs a simple iterative cycle until your evaluators pass or max attempts are exhausted.

01
📋

Spec

Define your goal and evaluation criteria

02
💡

Suggest

Your engine proposes a change

03
🔧

Apply

Patch is applied to the workspace

04
✅

Evaluate

Run tests, linters, type checks

05
🔁

Repeat

Loop until evals pass or max attempts

Quick start

Up and running in seconds

Bring your own suggestion engine. Use any shell command as an evaluator. BozoLoop handles the rest.

example.ts 
import { createLoop, commandEvaluator } from "bozoloop";

const result = await createLoop({
  goal: "All tests pass",
  workspace: ".",
  maxAttempts: 10,
  engine: {
    async suggest(ctx) {
      // Call your LLM, read files, generate diffs
      return {
        summary: `Fix attempt #${ctx.attempt}`,
        patch: { /* your patch data */ },
      };
    },
  },
  applier: {
    async apply(patch, workspace) {
      // Write files, apply diffs, run formatters
      return { ok: true };
    },
  },
  evaluators: [
    commandEvaluator("tests", "npm test"),
  ],
}).run();
Features

Minimal by design

Everything you need for iterative code improvement. Nothing you don't.

📦

Zero Dependencies

No runtime dependencies. Nothing to audit, nothing to break. Just TypeScript and Node built-ins.

📒

Full Ledger

Every attempt is recorded to disk with complete detail. No mystery behavior, no hidden magic.

⏸️

Pause & Resume

Stop the loop between attempts and pick up where you left off. State persists to disk.

⏪

Checkpoints

Snapshot the workspace before each attempt. Rollback to any checkpoint if things go wrong.

🔌

Lifecycle Hooks

Hook into every phase: start, suggest, apply, eval, end. Send webhooks, log, deploy previews.

🧩

Composable

Use from code, CLI, or wrap in your own automation. Bring your own engine and applier.

CLI

Run from your terminal

A dead-simple CLI with zero additional dependencies. Four commands, clear output, full state on disk.

bozoloop run

Start the loop. Iterate until evaluators pass or max attempts reached.

bozoloop resume

Pick up a paused loop from where it left off. State is read from disk.

bozoloop inspect

View current status, last attempt, and totals. Human-readable output.

Compatibility

Works with your tools

BozoLoop isn't trying to replace your editor or your favorite AI tool. It's a loop primitive you plug around them.

✓

AI Code Agents

Use Codex, Cursor, Claude, or any agent as your suggestion engine.

✓

Any LLM API

OpenAI, Anthropic, local models — bring whatever generates your patches.

✓

Existing Test Suites

Use commandEvaluator to run any CLI tool as an evaluator.

✓

Git & CI/CD

Works naturally with git workflows. Run BozoLoop in your pipeline.

✓

Any Editor

It's a library and CLI, not an IDE plugin. Works from any environment.

✓

Webhooks & Notifications

Use hooks to call external services, notify Slack, deploy previews.
